# Дорожная карта WB Analytics v2.0

## Правило 5 шагов (для каждой фазы)
1. `/discovery-interview` — проводим интервью, выясняем требования
2. Создаём ТЗ на основании данных
3. Реализуем по ТЗ
4. Тестирование на соответствие
5. `/debug` — отладка и приёмка

> Перед переходом на следующий шаг — получаем подтверждение.

### Фаза 15 — AI CFO Dashboard: Обзор + Поставки 🔴 Высокий
**Источник:** описание реализованного функционала конкурентного продукта (AI CFO)
**Зависимость:** Фазы 0, 2, 8

#### 15.1 — Вкладка «Обзор» (единая точка входа)

**Шапка страницы:**
- Переключатель периода (С начала года / произвольный диапазон)
- Поиск по SKU — глобальный фильтр
- Кнопка «Экспорт»
- Индикатор «Обновлено X мин назад · авто»

**4 карточки-сигнала (оперативные уведомления):**
- Продажи: заказы вчера + отклонение к аналогичному дню недели
- Реклама: кол-во задач по рекламе, из них критичных
- Поставки: SKU к заказу сейчас / на неделе
- Данные: замечания к качеству (нет себестоимости, предупреждения)

**8 KPI-карточек (ключевые финансовые метрики):**
- Реализация, Чистая прибыль (с маржой), Маржа %
- Выручка вчера (предв., с кол-вом заказов)
- ROI, % выкупа, Прибыль на единицу, Возвраты
- Отрицательные значения — красным, предварительные — бейдж «предв.»

**Блок инсайтов (автоматические выводы):**
- Главный убыток по SKU
- Лучший по прибыли SKU
- Лучший ROI SKU
- Итог возвратов + % от реализации
- Выкуп по кабинету

**3 карточки суточного мониторинга «вчера»:**
- Прибыль вчера (предв.)
- Расход на рекламу вчера (предв.)
- Критичных действий (охватывает N SKU)

**Водопад прибыли (Profit Waterfall):**
- Горизонтальные бары: Реализация → Себестоимость → Комиссия WB → Логистика → Чистая прибыль
- Индикатор сверки: «Реализация → выплата WB сверена»

**Мини-график «Заказы и продажи» (28 дней):**
- Двойной линейный: заказы (шт) + выручка (₽) с итогами

**Блок «Топ задач»:**
- Список приоритетных задач с переходом в полный список
- Сгруппированы по типу, привязаны к SKU/поставкам

#### 15.2 — Вкладка «Поставки»

**Шапка:** дата данных + индикатор актуальности + выгрузка XLSX/CSV

**3 KPI-карточки локализации (предв.):**
- Локализация % (доля товара в регионах продаж)
- Индекс КТР (коэффициент товарного распределения)
- Заказов учтено (база расчёта рекомендаций)

**Догрузка по округам (бар-чарт):**
- ЦФО / СФО+ДФО / ЮФО+СКФО / ПФО / СЗФО / УФО → шт к догрузке

**Таблица «Что и сколько допоставить»:**
- Артикул (название + nmId), ABC-класс, Остаток, Уже едет
- К отгрузке (предв.) с иконкой-подсказкой методологии
- Разбивка по округам с конкретным складом назначения (Электросталь, Коледино, Шушары, Казань, Краснодар, Новосибирск...)
- Сортировка по убыванию «К отгрузке»
- Фильтр по ABC-классу, поиск по артикулу/nmId

**Блок «Залежи» (неликвидные остатки):**
- Счётчик: N SKU · хранение за период X ₽
- Таблица: артикул, стоимость хранения (красным), заказы 28д (= 0)

**Дисклеймер:** предварительность расчётов до закрытия недельного отчёта

*Замечания:*
- Расчёт «К отгрузке» = (средний расход в день × (срок поставки + страховой запас)) − остаток − в пути
- Индекс КТР и Локализация % берутся из ИРП/аналитического API (wb_analytics_token)
- Залежи = товары с нулевыми заказами за 28д при положительном остатке
- Геораспределение по округам требует маппинга склада WB → федеральный округ

### Фаза 17 — Дата пустого склада + авто-задачи 🔴 Высокий

**Источник:** пользователь, логика управления запасами
**Зависимость:** Фазы 7, 8

- В справочнике по товарам (`/catalog`) добавить расчётное поле **«Дата пустого склада»**
- Расчёт: `текущий остаток (ВБ+ФФ) / скорость продаж (шт/день за последние 30 дн)` → дата = сегодня + дней_запаса
- Подсветка: красная если дата < 15 дней, жёлтая если < 30 дней
- **Авто-создание задачи** при дата < 15 дней:
  - Текст: `«Пустой склад раньше чем через 15 дней — [nm_id] — [vendor_code]»`
  - Прикрепить фото товара
  - Приоритет: critical
  - Дедупликация: не создавать если уже есть открытая задача с этим nm_id

### Фаза 22 — Платное хранение WB ✅ Выполнено

**Источник:** WB API `/api/v1/paid_storage` + использование в других блоках
**Цель:** точные данные по хранению с детализацией по SKU, складу и дате

**Таблица `wb_storage_daily`:** nm_id, date, warehouse, volume, cost, barcode, vendor_code
**Синхронизация:** ежедневный cron (0 6 * * *), окно 31 день за раз, история за 12 мес.
**Страница `/storage`:**
- KPI: хранение за период ₽, ср. хранение/шт/день, топ-дорогих SKU
- График хранения по дням (линия)
- Таблица SKU: хранение ₽/28д, хранение на шт, остаток, индекс = хранение/выручка
- Фильтры: период, склад
**Использование в других блоках:**
- Залежи (/supplies) → хранение берётся из `wb_storage_daily` вместо `wb_finance.storage_fee`
- дк-матрица → хранение/шт в строках матрицы
- P&L → статья «Хранение» точная по дате
- Обзор → KPI хранения за период
