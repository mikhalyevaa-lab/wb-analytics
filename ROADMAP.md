**Последнее обновление:** 2026-06-25  
**Версия документа:** 1.0 (объединяет v2.0, v3.0, v4.0)

---

## О проекте

**WB Analytics** — аналитическая платформа для продавцов на Wildberries. Подключается к WB API, загружает данные по заказам, продажам, финансам, рекламе и хранению, рассчитывает метрики и показывает их в едином дашборде.

**Текущая фаза:** MVP работает. Один магазин, один пользователь → переход к мульти-польз. командному доступу.  
**Горизонт:** SaaS-платформа для нескольких продавцов.

---

## Основные принципы

| Принцип | Суть |
|---------|------|
| **Data-first** | Каждая цифра верифицируется сверкой с официальными отчётами WB. Расхождение > 2% — баг. |
| **Правило 5 шагов** | Discovery → ТЗ → Реализация → Тестирование → Приёмка. Без явного подтверждения — не переходим. |
| **Прозрачность расчётов** | Пользователь видит откуда берётся каждая цифра. Hint-подсказки на всех ключевых полях. |
| **Защита данных** | Чувствительные данные (себестоимость, P&L, налоги) скрыты для ролей без доступа — и в UI, и на уровне API. |
| **Безопасность кода** | Нет `.env` в git. Нет `git push --force`. Миграции только добавляются, не удаляются. |

## 8. Хранение WB

**Статус:** ✅ Реализовано (данные Jan–Jun 2026)

### Что сделано
- **Страница `/storage`** — аналитика по платному хранению
- **KPI:** хранение за период ₽, среднее в день, среднее на шт
- **Таблица SKU:** хранение ₽/28д, хранение/шт, остаток, индекс хранение/выручка
- **Индекс хранение/выручка:** норма < 3%, красная зона > 10%
- **Данные:** `wb_storage_daily`, синхронизируется ежедневно в 6:00 МСК
- **Блок «Залежи»:** SKU с нулевыми заказами за 28 дней при ненулевом остатке — сколько денег «заморожено»

### Критическое замечание по данным
- API `paid_storage` (`warehousePrice`) даёт в **17–23× меньше** чем `wb_weekly_reports.storage_cost`
- Причина: `warehousePrice` — цена за единицу объёма, не включает НДС, доп.тарифы и операционные сборы
- **Вывод:** для финансовой аналитики (P&L) использовать `wb_weekly_reports.storage_cost`, не API
- Предупреждение пользователю показывается в UI
