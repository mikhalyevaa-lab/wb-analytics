## Про проект
WB Analytics — веб-приложение для аналитики магазина на Wildberries. Подтягивает данные через WB API (заказы, продажи, остатки, реклама, финансы), считает метрики (выручка, ДРР, ABC-анализ, воронка, юнит-экономика) и показывает их в удобном дашборде. Сейчас один пользователь — владелец магазина, в перспективе планируется расширение на других продавцов (SaaS).

## Стек
- **Frontend / API**: Next.js (App Router) + TypeScript + Tailwind CSS + shadcn/ui
- **База данных**: PostgreSQL 17 в Docker (данные в `./data/postgres`), миграции в `supabase/migrations/`
- **Auth**: Supabase Auth → планируется замена на `better-auth`
- **DB-клиент**: `@supabase/supabase-js` → планируется замена на `postgres.js`
- **Деплой**: Vercel (прод) + Docker Compose (локально / self-hosted)
- **Внешние API**: Wildberries Seller API (статистика, аналитика, реклама)
- **Cron**: Vercel Crons (прод) / отдельный Docker-контейнер (локально)

## Что нельзя
- Не коммитить `.env`, `.env.local` и любые файлы с секретами/токенами
- Не делать `git push --force` без явного подтверждения
- Не удалять миграции из `supabase/migrations/` — только добавлять новые
- Не менять `schema_complete.sql` напрямую — он обновляется вручную после миграций
